# OmniMentor — Mac Run & Verification Commands
Version: 1.1 (2026-03-07)

## Privacy/Safety reminder (HARD STOP)
This repo must remain organization-agnostic. Use synthetic-only data. Do not commit `.env`, db files, logs, or raw reports containing sensitive identifiers.

## Prerequisites
Required:
- macOS
- Git
- Node.js 20+
- pnpm
- sqlite3

Optional (later phases):
- Helm + Rancher Desktop
- Neo4j + Qdrant
- Ollama

## Install (Homebrew)
```bash
brew update
brew install git node pnpm sqlite
```

Optional later:
```bash
brew install helm
brew install --cask rancher
```

## Clone + install
```bash
git clone <YOUR_REPO_URL>
cd <YOUR_REPO_DIR>
pnpm install
```

## Run (dev)
### API
```bash
pnpm --filter @omnimentor/api dev
```
Verify:
```bash
curl -s http://localhost:3001/health
```

### Web
```bash
pnpm --filter @omnimentor/web dev
```
Verify: open the printed localhost URL and confirm scenario list loads.

## Phase 1 verification loop (daily “green”)
Run these after meaningful changes:
```bash
pnpm lint
pnpm test
pnpm typecheck
pnpm smoke
```

## Quality gates (baseline)
```bash
pnpm audit
```

## End-to-end smoke (Phase 1 gate)
```bash
pnpm smoke
```
Expected (Phase 1):
- Fetch scenarios
- Create a submission
- Score it (gating + rubric)
- sqlite rows exist
- report written under `reports/` (week folder if present)

## sqlite quick checks
Use the DB path documented in `PROJECT_CONTEXT.md`.
```bash
sqlite3 <DB_PATH> ".tables"
sqlite3 <DB_PATH> "select count(*) from submissions;"
sqlite3 <DB_PATH> "select count(*) from score_reports;"
```

## Reports
```bash
ls -la reports
ls -la reports/week1 2>/dev/null || true
```

## Optional later: integration + UI E2E tests
Add these once the API/UI stabilize (avoid brittle tests early):
```bash
pnpm test:integration
pnpm test:e2e
```

## Verification log (required)
Create and update `VERIFICATION_LOG.md` in the repo root after each session with:
- date/time
- commands run
- observed results
- smoke output filename
- report paths
